开发规范

开发规范

  1. 在开始之前，明确自己是想要 维护网站内容，还是要 改进网站的功能特性；
  2. 任何更改都应该发生在dev分支上，不要手动修改master分支；
  3. 源码缩进 一定 要统一，推荐缩进为2个空格，不要 用 TAB 缩进；
  4. 开发网站的时候，记住 模版优先 和 移动端优先 两个重要概念；前者可以大量减小工作量，提升网站可维护性；后者保证移动端访问网站时的友好性；
  5. 降低页面之间的耦合性；
  6. 推送之前，一定要先在本地测试，并且功能更新代码 必须 完善之后才能推送；
  7. 自动触发远程编译是用来 部署 的，不是 用来测试的。想及时看到修改效果，最好的方法是在本地编译测试。
  8. 框架可以自由扩展，但是 必须 保持易读性和整洁性。
  9. 所有用到的网页组件，参考 Bootstrap Material Design 页面的组件实现。点击各个组件示例旁边的 <> 可以显示该部分的源代码。
